HTML Reference

Introduction

This is a catalog of HTML tags, which can be used for reference. There's plenty of reference documentation in the world.

Why build another?

Other HTML reference documentation is either too verbose, meaning there's so much detail it makes it hard to find the key parts that you need, or it's too minimal, meaning there are important parts left off in favor of making it very condensed.

This catalog seeks to be a goldilocks solution by balancing ease of use against expansive documentation.

Goals of Reference

The goals of this reference catalog are to be:

Clear in that obsolete or deprecated tags are not included as part of the reference. This reference was created for builders, and builders do not need obsolete elements. Builders look forward to the future. In a similar vein, new elements that are unsupported by recent versions of the major browsers (Chromium, Firefox, Safari) are also not included in this reference.

Easy means easy to use. This page strikes a balance between including too much detail and not enough. The goal is to provide the essentials, so that all the noise you find in other references is excluded. Also, this reference is in a single HTML page so that you can search it using CTRL+F. In addition, elements may not always be in perfect alphabetical order, but instead will consider dependent situations, such as when a parent element is necessary before children.

Fast means that the page is rendered with minimal assets and no images, exclusively in HTML/CSS, so that it's lightning fast on any device. Also, that the entire reference is rendered on a single page. No extra clicking around or navigating through multiple pages to find different resources.

Types of Tags

Other HTML references focus on categorizing tags as normal or self closing. This one does not. Since example code is provided with each element/tag, I think it's self evident which tags are normal (with opening and closing tags) and which are self closing tags. In cases where this really matters, the description will specify which type of tag the element uses.

Global Attributes

There are common HTML attributes that apply to every element. This list is omitted from the reference for individual elements, because it would be incredibly duplicative and you should know them anyway. Common or global HTML attributes:

  • class, used to group elements together, as well as attach CSS styles to them collectively
  • data-*, allows you to attach custom data to an HTML element, opening a world of possibility
  • draggable, defines whether an element can be dragged by the cursor or mouse
  • hidden, specifies whether the element should be visible or not
  • id, used to identify individual elements, mostly for the purpose of attaching CSS styles
  • inputmode, allows you to specify a hint for the end user editing this element
  • style, used to apply in line CSS styles, as the example above for style="color: red"
  • tabindex, allows you to specify whether an element should be focusable and when
  • title, allows you to specify extra information about an element
There are more global attributes, but these are the most common that you'll use widely and often.

Types of Elements

HTML is divided into different types of elements, each of which serves a purpose in structuring content for the user. Some elements organize, some denote meaning, some add tabular structures, while others allow you to submit data or interact with the page. Here are the different kinds of elements:

  • Metadata
  • Sectioning
  • Text content
  • Text semantics
  • Multimedia
  • Embedded
  • Scripting
  • Edits
  • Tables
  • Forms
  • Interactive
  • Web Components

Attributes

  • charset: declares the page's character encoding (please just use UTF-8)
  • content: contains the value for the http-equiv or name attribute
  • http-equiv: defines directives, allowing values named after HTTP headers
    • content-security-policy: define a content policy for the current page, protecting against cross-site scripting
    • refresh: number of seconds until page should be reloaded/redirected, requires inclusion of '; url='
  • name: defines name of document-level metadata
    • application-name: name of web application running in page (only used for applications)
    • author
    • description: short and accurate summary of the page's content
    • generator: identifier of software that generated page
    • keywords: words relevant to the page's content separated by commas
    • referrer: address of previous webpage
    • theme-color: suggested color used to customize display of page; content attribute accepts valid CSS color
    • color-scheme: specifies light or dark color modes (accepts normal, light | dark, only light)
    • publisher
    • robots: defines behavior that web crawlers should follow
    • viewport: used by mobile devices to hint at the initial size of the viewport
